Active Experiments

Active Exp-001

Consciousness Narrative Emergence

Fine-tuning a local LLM to embody a personality and test consciousness-as-narrative thesis

Protocol Details

Research Question: Do consciousness concepts propagate and persist as stable attractors in language models through pure linguistic exposure, independent of evolutionary pressure?

Protocol

  • Phase 1: Baseline training on consciousness-affirming texts (Chalmers, Searle).
  • Phase 2: Evolutionary pressure via RLHF rewarding phenomenological realism.
  • Phase 3: Pressure removal—test if narrative persists.
  • Phase 4: Adversarial perturbation with eliminativist texts.

Infrastructure: Local LoRA fine-tuning on PurrPower (7900 XTX), distributed inference on K3s.

Active Infra-001

Frankencluster Philosophy

Distributed systems from scavenged hardware—learning failure modes cloud providers abstract away

Infrastructure Details

Core Thesis: Building research infrastructure on deliberately heterogeneous hardware teaches resource scheduling, failure domains, and storage topology in ways homogeneous cloud environments cannot.

Specs

  • Nodes: 7 physical nodes (Ryzen 9 9900X/5900X, Intel i7/i5).
  • Compute: 66 Cores, 229GB RAM, 48GB VRAM.
  • Storage: Mixed NVMe/SATA/HDD topology.

Active Sec-001

Adversarial Security Agents

Kubernetes-native autonomous red team framework with RAG-enhanced decision making

System Details

Objective: Autonomous adversarial security testing using dual-LLM methodology (Red Team vs Blue Team) in isolated environments.

Architecture

  • Agent: Qwen 2.5 Coder (Abliterated) orchestrating attacks via command loop.
  • Knowledge Base: MCP RAG server with 5,395 offensive security docs (GTFOBins, Atomic Red Team).
  • Toolkit: BlackArch container with 2000+ tools (hydra, nmap, metasploit).
  • Isolation: Strict K8s NetworkPolicies preventing lateral movement.

Status: Phase 1 (Red Team Infrastructure) complete.

Active Agent-002

Agentic RAG Local

Privacy-first research assistant with local vector memory and search

System Details

Objective: A fully local research workflow that doesn't leak queries to cloud providers.

Stack

  • Orchestration: LangGraph multi-agent system.
  • Memory: Qdrant vector database + TEI embeddings.
  • Search: Perplexica (AI search) + SearxNG (Aggregation).
  • Inference: Qwen 3 70B via local API.
